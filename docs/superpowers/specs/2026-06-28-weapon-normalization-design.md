# 猪猪黄昏 武器正常化设计

- 日期：2026-06-28
- 主题：武器视觉与判定正常化
- 范围：移除夸张武器，统一剩余武器的手持表现与攻击距离

## 1. 背景

当前武器系统已经支持多把武器切换，但武器素材尺寸、握把锚点和通用挥舞动画之间存在明显失配，导致部分武器出现以下问题：

- 武器离手过远，像是漂浮在角色前方
- 武器面积过大，视觉长度远超角色体型
- 视觉长度与实际攻击距离不一致
- 少数夸张武器破坏整体美术与手感统一性

本次调整优先解决这些直接影响观感和手感的问题，不引入新武器系统，不改动现有 HUD、掉落、升级或关卡结构。

## 2. 目标

本次改动的目标如下：

- 删除流星锤与枪，不再出现在武器池、选择界面或运行时装备流程中
- 将所有剩余武器收敛为“正常手持比例”
- 让每把武器在静止和攻击时都更贴手，看起来确实由屠夫握持
- 收紧攻击范围，使视觉长度与命中距离基本一致
- 保持现有武器稀有度、描述、攻击类型和基础玩法差异

本次不做：

- 新增武器
- 重做武器系统架构
- 修改敌人、地图或主动技能逻辑
- 为每把武器新增独立攻击判定形状

## 3. 方案对比

### 方案 A：只改视觉表现

- 删除流星锤与枪
- 统一缩小武器 worldScale
- 调整 grip 与通用攻击动作

优点：实现快，改动少。

缺点：如果保留原 range，仍可能出现“武器看起来变短了，但攻击距离还很远”的失配。

### 方案 B：视觉与判定一起收紧

- 删除流星锤与枪
- 调整每把武器的 worldScale、grip、持握偏移和挥动幅度
- 同步下调每把武器的 range

优点：视觉和手感更统一，问题解决更完整。

缺点：需要回归测试攻击手感和武器选择流程。

### 结论

采用方案 B。因为当前问题不只是“图太大”，而是表现和命中距离同时失配，只改一层会留下明显违和感。

## 4. 设计细节

### 4.1 武器池调整

从 `weaponCatalog` 中删除以下武器：

- `pigHeadFlail`
- `sausageCannon`

删除后，相关界面与运行时逻辑应通过已有的目录驱动行为自动收缩，不保留占位卡片或隐藏条目。

### 4.2 视觉正常化

对所有保留武器执行以下约束：

- `worldScale` 收敛到接近屠刀的正常区间
- 不再保留任何“巨型武器”专用尺寸
- `grip` 重新校准到真实握持位置
- 挥舞时的 `reach` 前伸距离缩短
- `rotation` 幅度收窄，避免武器绕角色大范围甩出

视觉上允许武器之间保留轻微大小差异，但不允许再出现超出角色体型数倍的表现。

### 4.3 判定正常化

对所有保留武器同步调整：

- 下调 `range`
- 保持武器之间的强弱梯度，但整体区间收紧
- 避免“短柄武器打出远距离命中”的观感

本次不引入复杂扇形判定或投射物系统，仍沿用当前的最近敌人自动攻击模型。

### 4.4 动画表现

保留现有按 `attackProfile` 区分动作的方式，但调整对应幅度：

- `slash`：小幅前送与较短挥砍
- `heavy`：比普通挥砍更重，但不再有大开大合的夸张位移
- `clamp`：保留机械感，但贴近角色手部

删除 `flail` 与 `shoot` 对应的特殊使用场景后，攻击动画分支应仍然稳定工作，不产生缺失纹理或错误动画引用。

## 5. 影响范围

预计涉及：

- `src/game/data/weaponCatalog.ts`
- `src/game/entities/Player.ts`
- `src/game/scenes/BootScene.ts`
- `src/game/scenes/WeaponSelectScene.ts`
- 相关测试文件

数据驱动部分应继续保证：

- 未知武器 id fallback 到起始武器
- 武器选择界面正常展示剩余武器
- 武器贴图加载表与目录一致
- 运行时装备已存在武器 id 时不会崩溃

## 6. 测试策略

需要补充或更新以下验证：

- 武器目录测试：删除的武器不再存在
- 武器尺寸测试：不再允许“巨型武器” scale 断言
- fallback 测试：非法 id 仍回退到 `rustyCleaver`
- 构建与测试：`npm test`、`npm run build`

如条件允许，再进行一次本地试玩确认：

- 武器选择界面显示正常
- 屠刀、大剑、钩类和夹具类都贴手
- 自动攻击时可视距离与命中距离没有明显分离
