# 猪猪黄昏 设计文档

- 日期：2026-06-27
- 项目名：猪猪黄昏
- 类型：俯视角单图波次制爽感割草肉鸽
- 当前目标：定义首个可试玩的小型完整 demo 范围与实现方向

## 1. 设计目标

《猪猪黄昏》是一款以荒诞屠宰场为题材的俯视角割草肉鸽。玩家扮演屠夫本人，在固定屠宰场地图中迎战一波又一波荒诞化猪潮。项目的第一优先级不是复杂经营、长线数值或重剧情，而是做出强烈且稳定的单局爽感。

首个 demo 应该让玩家在短时间内明确感受到以下体验：

- 进入战斗后 30 秒内就能理解玩法并开始爽起来
- 自动攻击持续清杂，主动技能负责爆发、救场和 build 亮点
- 升级频繁，玩家能明显感受到自己越来越像“屠宰机器”
- 题材记忆点强：土味、锈蚀、荒诞、黑色幽默，但不走怪物化恐怖路线

## 2. 产品定位

### 2.1 核心定位

- 核心方向：爽感割草
- 战斗结构：自动攻击清杂兵 + 手动释放强技能
- 角色身份：屠夫本人
- 敌人方向：普通猪的荒诞化，不做异变怪物
- 地图结构：固定屠宰场地图
- 单局目标：清空全部波次后击败最终猪王
- 版本范围：小型完整 demo

### 2.2 不做什么

为了避免首版失焦，以下内容不进入首个 demo 的必做范围：

- 多角色职业系统
- 多张地图
- 重经营或基地建设
- 复杂剧情分支
- 大规模局外成长
- 开放探索或半开放区域推进

## 3. 核心循环

单局核心循环如下：

1. 玩家进入固定屠宰场战场。
2. 猪潮从多个入口按波次涌入。
3. 屠夫依靠自动主武器持续清理普通猪群。
4. 玩家在高压时机手动使用主动技能脱困、控场或打爆发。
5. 击杀敌猪后获得经验与金币，并以较高频率触发升级选择。
6. 每过几波，敌人数量、组合、速度和压迫感逐步提升。
7. 中后段出现精英猪与小 Boss 级压力点。
8. 清空最终波次后进入猪王战，击败 Boss 即通关。

该循环的设计目标是：

- 始终有怪可打，保持屏幕忙碌感
- 玩家操作负担适中，不因纯手动输出而打断割草流畅度
- 每隔较短时间就有一次“变强反馈”
- 终局有明显收束，而不是无限拖时间

## 4. 角色、武器与技能结构

### 4.1 角色策略

首版 demo 固定单主角：屠夫本人。

不引入职业分化，把内容集中在同一个角色的战斗成长线上。屠夫的人设应天然支撑全部武器、道具和升级命名，例如屠刀、盐袋、铁钩、火把、猪圈、开膛、放血、腌制等题材语义。

### 4.2 三层战斗结构

战斗系统采用三层结构：

1. 自动主武器
2. 主动技能
3. 被动强化 / 遗物 / 升级词条

#### 自动主武器

默认主武器为屠刀，负责持续输出与清理杂兵。它必须稳定、直接、反馈明确，是玩家割草爽感的底盘。

建议表现：

- 近距离自动挥砍
- 半圆或短距扇形攻击
- 命中带轻微击退与清脆打击感

#### 主动技能

主动技能承担操作亮点与危机处理，不宜做得太多，但必须足够有辨识度。首版建议固定实现 3 个核心技能：

- 屠刀旋斩：近身范围清场，适合被包围时解围
- 盐袋爆撒：前方范围爆发，适合打密集猪群
- 猪圈陷阱：地面控场技能，限制冲锋与聚团敌人

#### 被动强化

被动强化决定每局 build 的变化方向。首版聚焦直观且即时生效的强化，避免大量长线规划型词条。

强化方向可包括：

- 攻击力
- 攻速
- 暴击率
- 暴击伤害
- 攻击范围
- 击退
- 流血 / 灼烧
- 主动技能冷却缩减
- 主动技能额外效果
- 移速
- 掉落收益

## 5. 敌人、波次与 Boss

### 5.1 敌人基调

敌人基调定为“普通猪的荒诞化”。它们仍然是猪，但行为、装备和功能会越来越离谱。这样能保持题材统一，并强化黑色幽默，而不是把体验导向怪物猎杀。

### 5.2 首版敌人池

首版 demo 目标为 8 到 10 种敌猪，其中建议优先实现以下核心类型：

- 肥猪：高血慢速基础压力单位
- 瘦猪：高速脆皮追击单位
- 叉猪：中距离突刺单位
- 头盔猪：高护甲抗压单位
- 冲锋猪：蓄力后直线冲撞
- 饲料猪：吸引附近猪群聚团，制造局部拥堵
- 精英屠栏猪：更大体型、更多血量、带特殊词缀

### 5.3 波次节奏

建议单局为约 10 波的清场结构：

- 第 1-3 波：教学与上手，熟悉走位、自动输出和主动技能
- 第 4-6 波：敌人组合复杂化，引入功能型敌猪
- 第 7-9 波：高压敌群、精英猪、入口联动、多方向压迫
- 第 10 波：终局大波次或猪王战前总压制
- 最终战：猪王 / 种公猪霸主

### 5.4 最终 Boss

最终 Boss 不追求复杂机制堆叠，而是强调“给整局养成一个足够大的出气筒”。建议 Boss 采用 2 到 3 个清晰机制：

- 大范围冲锋碾压
- 召唤小猪或精英猪
- 震地击退或区域踩踏

Boss 战目标是验证玩家这一局 build 的完整度，而不是突然切换成高门槛动作游戏。

## 6. 升级、经济与单局成长

### 6.1 成长原则

成长系统的关键词是“频繁”。玩家需要不断收到明确的增强反馈，而不是长时间等待数值兑现。

### 6.2 三条成长线

#### 经验升级

击杀敌猪获得经验，升级时从 3 个选项中选择 1 个。选项来源分为：

- 主武器强化
- 主动技能强化
- 通用被动强化

#### 金币积累

敌猪掉落金币。首版金币主要承担轻量功能，例如：

- 升级选项重置
- 波次间小补给
- 少量临时增益

不建议首版引入复杂商店树和大规模局内交易系统，以免割裂战斗节奏。

#### 稀有掉落

低概率掉落临时功能物，作为惊喜反馈，例如：

- 临时回血
- 短时狂暴
- 一次性爆炸物
- 特殊主题道具

### 6.3 升级词条原则

升级词条应尽量做到“看一眼就懂，拿了立刻变强”。首版词条示例：

- 屠刀淬炼：攻击力提升
- 开膛熟练：暴击率提升
- 粗盐增量：盐袋范围扩大
- 猪圈加固：陷阱持续时间增加
- 火候正好：主动技能附带灼烧
- 血气上涌：低血量时提升攻速

### 6.4 局外成长策略

首版局外成长应保持极轻量。可以先只做：

- 图鉴
- 结算记录
- 通关统计

不做大量永久成长数值树，以免模糊 demo 的核心验证目标。

## 7. 地图与场景设计

### 7.1 地图结构

地图固定为单张屠宰场战场，作为首版 demo 的唯一主战场。其意义是把开发资源集中投入到：

- 战斗手感
- 刷怪节奏
- 升级反馈
- 题材风格

而不是分散到多图探索和地图切换系统。

### 7.2 地图分区

虽然是单图，但可在视觉和战术上划分子区域，例如：

- 猪圈
- 仓库
- 饲料堆放区
- 冷库外场
- 泥泞屠宰空地

这些区域主要提供：

- 刷怪入口差异
- 地形绕行
- 陷阱和障碍布局
- 视觉识别度

### 7.3 场景制作策略

地图美术生产优先使用 `generate2dmap` skill。

在本项目中，地图建议采用：

- `map_mode`: `scene_mode`
- `visual_model`: `layered_raster`
- `runtime_object_model`: `separate_props` 或 `y_sorted_props + interactive_scene_objects + scene_hooks`
- `collision_model`: `precise_shapes + trigger_zones`

原因：

- 游戏是俯视角固定单图 arena，不需要 tile 编辑优先
- 需要强风格基础场景
- 需要将障碍物、陷阱和互动对象与底图分离
- 便于后续接入碰撞、刷怪点和场景事件

地图生成时应遵守以下约束：

- 基础底图只保留地面、道路、污渍、墙基、地块边界等稳定元素
- 可碰撞物、围栏、桶、灯、招牌、火盆、猪圈门、陷阱等必须拆成独立运行时对象
- 产出应包含底图、对象摆放元数据、碰撞数据和 QA 预览

## 8. 角色、敌人和技能资产生产

可见角色与技能资产优先使用 `generate2dsprite` skill 生成，不使用程序临时画图替代正式可试玩内容。

### 8.1 首版重点资产

建议优先生成以下资源：

- 屠夫主角 sprite
- 肥猪、瘦猪、叉猪、头盔猪、冲锋猪等敌猪 sprite
- 猪王 / 最终 Boss sprite
- 主武器攻击表现
- 屠刀旋斩、盐袋爆撒、猪圈陷阱的技能特效
- 受击 / 死亡 / 掉落反馈素材

### 8.2 风格要求

所有 sprite 应匹配地图风格。优先采用：

- `art_style`: `map_style` 或 `project-native`
- 目标观感：脏旧、厚重、荒诞、夸张、可读性强

若地图采用 clean HD 的脏旧绘制风格，则 sprite 也应沿用相同材质语言，而不是切成纯复古像素风。

## 9. UI 与风格方向

### 9.1 视觉方向

UI 与整体包装应围绕以下关键词：

- 黄昏
- 铁锈
- 屠宰场
- 土味招牌
- 黑色幽默
- 旧式操作台

参考图所展示的“铁框包边 + 暖红暮色 + 厚重木铁材质 + 信息密集但清晰”的方向是正确的。

### 9.2 首版必需 HUD

首版必须包含以下 HUD 模块：

- 生命
- 波次
- 时间
- 金币
- 技能栏
- 升级弹窗
- 简化任务 / 目标提示
- 结算界面

### 9.3 首版不优先项

以下 UI 可弱化或延后：

- 复杂背包系统
- 深层图鉴交互
- 大量二级菜单
- 多页面成长界面

## 10. 任务目标与通关结构

首版 demo 的目标设计应简洁明确。

### 10.1 主目标

- 进入屠宰场
- 清空全部波次
- 击败猪王

### 10.2 可选附加目标

为增加 replay，可以加入少量附加挑战，但它们不应阻碍主循环：

- 本局击杀指定数量的猪
- 不使用陷阱通关
- 在某一波前保持高血量
- 使用某个技能造成指定伤害

这些目标主要作为结算奖励、挑战文本或图鉴记录，而不是剧情推进系统。

## 11. Demo 范围锁定

首个小型完整 demo 的最低可交付内容锁定为：

- 1 张固定屠宰场地图
- 1 名主角：屠夫
- 1 把自动主武器：屠刀
- 3 个主动技能
- 8-10 种敌猪
- 1 个最终 Boss
- 1 套完整单局升级池
- 完整 HUD
- 升级选择界面
- 结算界面

达到以上内容时，demo 就具备以下价值：

- 能验证题材是否成立
- 能验证战斗是否爽
- 能验证美术和 UI 是否有记忆点
- 能拿给外部玩家试玩并获取真实反馈

## 12. 测试与验收标准

首版 demo 的验证重点应放在玩家主观体验与系统清晰度上。

### 12.1 功能验收

- 单局可从开始打到通关或失败
- 波次能正常推进
- 主武器自动攻击稳定工作
- 3 个主动技能都可用且有明确差异
- 升级选择可稳定出现并生效
- 敌猪种类能体现不同压力
- 最终 Boss 可正常结算

### 12.2 体验验收

- 玩家在前 1 分钟内能理解玩法
- 前 3 分钟内能感受到第一次明显 build 成型
- 中后期屏幕压迫感足够，但不是纯粹数值碾压
- UI 虽然信息多，但不会看不懂
- 题材风格能让试玩者在结束后记住“这是那个杀猪肉鸽”

## 13. 后续规划边界

如果首版 demo 验证成功，下一阶段可以考虑扩展：

- 新地图
- 新主武器
- 新技能流派
- 新精英猪和 Boss
- 更完整的局外成长
- 剧情事件和荒诞世界观推进

但这些全部属于下一阶段，不进入当前 spec 的必做范围。
